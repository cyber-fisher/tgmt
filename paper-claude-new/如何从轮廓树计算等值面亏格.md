# 如何从轮廓树计算等值面亏格

基于 Morse 理论、Euler 示性数与轮廓树增强标注的落地方案。

> 结论先行：仅凭“轮廓树的连接形状”无法唯一推出等值面的亏格。可落地做法是把轮廓树作为等值面连通分量的演化骨架，再在构树或遍历过程中为每条弧补充 Euler 示性数 $\chi$、连通分量数 $c$、弧上等值面样本或临界点贡献等辅助信息，最后用 $g = c - \chi / 2$（闭合可定向曲面）或逐分量 $g_k = 1 - \chi_k / 2$ 得到亏格。

## 1. 问题定义：要计算什么

输入是定义在三维体数据或四面体网格上的标量场：

$$
f: M \rightarrow \mathbb{R}
$$

以及已经构建好的轮廓树 `CT`。对任意一个等值 $h$，等值面：

$$
S_h = f^{-1}(h)
$$

可能包含多个不相交的闭合曲面分量。目标不是只判断“有没有孔”，而是为轮廓树上的每条弧 `e` 标注：

- 该弧对应的等值面连通分量或分量集合；
- 该弧内部任意正则等值 $h$ 的 Euler 示性数 $\chi(e)$；
- 连通分量数 $c(e)$；
- 亏格 $g(e)$，以及必要时的逐分量亏格列表。

在医学 CT、材料孔隙、流体模拟等数据里，$h$ 沿标量值变化时，只有穿过临界点时拓扑才会变化；在同一条轮廓树弧内部，等值面拓扑保持同胚，因此亏格保持不变。

## 2. 数学关系：亏格、Euler 示性数与连通分量

对一个闭合、可定向、无边界的连通二维曲面，Euler 示性数与亏格满足：

$$
\chi = V - E + F = 2 - 2g
$$

因此：

$$
g = \frac{2 - \chi}{2}
$$

若等值面包含 $c$ 个互不相连的闭合曲面分量，总 Euler 示性数为各分量之和：

$$
\chi_{total} = \sum_k \chi_k
$$

总亏格为：

$$
g_{total} = \sum_k g_k = c - \frac{\chi_{total}}{2}
$$

工程实现中最稳妥的换算方式是：先得到每个正则等值或每条轮廓树弧的 $\chi$ 与 $c$，再换算亏格。

如果表面有边界、非定向、网格破洞或提取算法产生非流形边，上式需要先修补，或改用带边界公式：

$$
\chi = 2c - 2g - b
$$

其中 $b$ 是边界环数量。

## 3. 为什么轮廓树本身不够

轮廓树压缩记录的是等值面连通分量随标量值的合并、分裂、诞生和消亡。它能告诉我们“哪个等值面分量连接到哪个分量”，但通常不会保留等值面内部是否绕出把手的信息。

- 两个不同标量场可能有相同的轮廓树结构，但某些弧对应的等值面一个是球面、另一个是环面。
- 轮廓树节点能提示拓扑事件发生的位置，不能单独给出事件对 $\chi$ 或 $g$ 的净贡献。
- 要计算亏格，必须在构树时保留临界点类型、局部上下链接，或在弧上抽样等值面并计算网格 Euler 示性数。

因此，“从轮廓树计算亏格”的准确表述应是：从增强后的轮廓树，或从“轮廓树 + 原始标量场/网格”，计算每条弧对应等值面分量的亏格。

## 4. 可落地算法方案

### 方案 A：弧上抽样等值面，直接算 $\chi$

这是最稳、最容易实现的方案。

1. 对每条轮廓树弧 $e = (u, v)$，选择一个正则等值 $h$，满足 $f(u) < h < f(v)$，且不等于任何顶点标量值。
2. 用 Marching Cubes、Flying Edges 或 TTK/VTK 的等值面提取算法生成 $S_h$。
3. 按连通分量分割三角网格，统计每个分量的 $V$、$E$、$F$。其中 $E$ 要按无向唯一边计数。
4. 计算 $\chi_k = V_k - E_k + F_k$，$g_k = 1 - \chi_k / 2$。
5. 把结果写回轮廓树弧属性：`sample_iso`、`component_count`、`euler_char`、`genus_list`、`genus_total`。

优点：鲁棒、容易验证，不需要完全解析所有临界点类型。

缺点：每条弧都要抽样提面，耗时较高；如果体数据很大，可只对候选弧或简化树后的弧计算。

### 方案 B：构树时累加临界点贡献

这是最快但实现要求更高的方案。

1. 对标量场进行 Simulation of Simplicity 或符号扰动，保证临界值全序且临界点尽量 Morse 化。
2. 对每个临界点计算上下链接（lower link / upper link），识别 minimum、maximum、1-saddle、2-saddle，以及 join/split 事件。
3. 沿标量方向扫掠轮廓树，在弧属性中传播 $\chi$ 与 $c$。
4. 普通弧内部 $\chi$ 不变；穿过临界点时按局部事件更新。
5. 在合并/分裂鞍点处，同时维护分量身份，避免只累加一个全局 $\chi$ 导致无法得到逐分量亏格。
6. 对每条弧用 $g = c - \chi / 2$ 输出总亏格；如果保存了分量级 $\chi_k$，则输出逐分量亏格。

方案 B 适合嵌入自研轮廓树构建器或 TTK 管线中，但必须严肃处理 1-saddle/2-saddle、非 Morse 多重鞍点、边界和非流形情况。

## 5. 临界点贡献表

| 事件 | 轮廓树表现 | 扫掠含义 | 对 $\chi/c$ 的典型影响 | 亏格影响 |
|---|---|---|---|---|
| Minimum | 底部叶子 | 新分量诞生 | $c + 1$，$\chi + 2$（闭合小球刚形成后） | $g = 0$ |
| Maximum | 顶部叶子 | 分量消亡 | $c - 1$，$\chi - 2$ | 移除该分量 |
| Merge/Split saddle | 内部度数节点 | 两个分量合并或一个分量分裂 | $c$ 改变 $\pm 1$；$\chi$ 按局部手柄/分裂事件更新 | 可能不变，也可能改变 |
| Handle saddle | 内部度数节点 | 同一分量长出或闭合把手 | $c$ 不变，$\chi - 2$（长把手）或 $\chi + 2$（闭合把手） | $g + 1$ 或 $g - 1$ |

注意：源材料中把 minimum 贡献写成 `+1`、1-saddle 写成 `-1`、2-saddle 写成 `+1` 的说法，更接近对“下水平集/胞腔”Euler 特征的 Morse 指数交替贡献，而不是闭合等值面自身的直接 $\chi$。若目标是等值面亏格，推荐以等值面网格 $\chi$ 或严格的局部等值面事件公式为准。

## 6. 实现细节与伪代码

推荐先实现方案 A，形成可验证的基线，再把热点弧替换为方案 B 的增量传播。

```python
for arc in contour_tree.arcs:
    h = choose_regular_value(arc.low.value, arc.high.value)
    mesh = extract_isosurface(volume_or_tetmesh, h)
    comps = connected_components(mesh)

    arc.euler = 0
    arc.component_count = len(comps)
    arc.genus_list = []

    for comp in comps:
        V = count_unique_vertices(comp)
        E = count_unique_undirected_edges(comp)
        F = count_faces(comp)

        chi = V - E + F
        genus = 1 - chi / 2

        arc.euler += chi
        arc.genus_list.append(genus)

    arc.genus_total = sum(arc.genus_list)

    # 对闭合可定向曲面，也可使用：
    # arc.genus_total = arc.component_count - arc.euler / 2
```

关键实现点：

- 唯一边计数：三角形 `(a, b, c)` 贡献 `(min(a,b), max(a,b))`、`(min(b,c), max(b,c))`、`(min(c,a), max(c,a))` 三条无向边，放入 `set` 去重。
- 正则值选择：取弧两端标量中点；若中点恰好等于某顶点值，则加入极小 `epsilon` 偏移。
- 弧与等值面分量映射：如果抽样得到多个分量，需要用轮廓树的分量标识或空间连通分割确认哪些分量属于该弧。
- 浮点鲁棒性：等值面提取前可对标量做稳定排序；避免 $h$ 落在平台值或多顶点相同值上。

## 7. 非 Morse 情况和工程边界

- 非 Morse 临界点：真实体数据常出现多个邻接单元同值或高阶鞍点。应先用 SoS、微扰、jitter 或稳定 tie-break 将其拆为一串二元事件。
- 边界数据：如果标量场定义在有边界区域，等值面可能有开口，需要统计边界环 $b$，用 $\chi = 2c - 2g - b$ 换算。
- 非流形网格：Marching Cubes 的歧义面、重复顶点、T-junction 会污染 $V/E/F$，应先 weld 顶点并检查每条边邻接面数量。
- 简化轮廓树：拓扑简化会删除小持久性分支，弧亏格也应随简化策略重新合并或重新抽样计算。
- Jacobi / Reeb 图差异：在含环定义域上，Reeb 图比轮廓树更一般；如果定义域本身不是 simply connected，轮廓树假设要重新检查。

## 8. 最小验证用例

| 数据/等值面 | $\chi$ | $c$ | 期望亏格 |
|---|---:|---:|---:|
| 单球面 | 2 | 1 | 0 |
| 两个不相交球面 | 4 | 2 | 0 + 0 = 0 |
| 单环面/甜甜圈 | 0 | 1 | 1 |
| 双环面 | -2 | 1 | 2 |
| 球面 + 环面 | 2 + 0 = 2 | 2 | 1 |

验收建议：先人工构造这些隐式场或网格，确认提取、连通分割、$V/E/F$ 计数和 $g$ 换算全部正确，再接入真实轮廓树。

## 9. 推荐落地路线

1. 第一阶段：保留现有轮廓树构建流程，对每条弧抽样等值面并直接计算 $V/E/F$，得到可验证的 `genus` 标签。
2. 第二阶段：将 `genus` 标签与弧持久性、弧长度、体积、面积等指标一起输出，用于筛选“拓扑复杂但体积小”的噪声结构。
3. 第三阶段：在性能成为瓶颈后，才引入临界点增量传播；用第一阶段结果作为回归测试。
4. 第四阶段：把非 Morse 拆解、边界环统计、网格流形检查做成独立 QA 模块，避免亏格标签被坏网格污染。

› 轮廓树中如果一个鞍点是k=2，l=2呢


• k=2, l=2 是一个非 Morse 的“混合鞍点”：从低到高看，两个分量在这个标量值处合并，同时又分裂
  成两个分量。标准 Morse 鞍点不允许一次同时发生两件事。

  SoS 拆法
  按前面的规则：

  (k - 1) 个 1-saddle + (l - 1) 个 2-saddle

  所以：

  k=2, l=2
  → 1 个 1-saddle + 1 个 2-saddle

  结构上可写成：

  e1 ─┐
      ├─ S1: 1-saddle ─ m ─ S2: 2-saddle ─┬─ u1
  e2 ─┘                                    └─ u2

  其中 m 是虚拟中间边。

  Euler / 亏格贡献
  如果用 q = χ / 2：

  1-saddle: Δq = -1
  2-saddle: Δq = +1

  所以总贡献：

  Δq = -1 + 1 = 0

  如果用完整 Euler 示性数 χ：

  Δχ = -2 + 2 = 0

  也就是说，k=2,l=2 这个节点整体不改变总 Euler 示性数，但它会重配连通关系。

  亏格传播
  设下方两条边的半 Euler 值为：

  q(e1), q(e2)

  先经过 1-saddle：

  q(m) = q(e1) + q(e2) - 1

  再经过 2-saddle：

  q(u1) + q(u2) = q(m) + 1

  合起来：

  q(u1) + q(u2) = q(e1) + q(e2)

  所以只能确定上方两条边的 q 总和，不能仅靠轮廓树结构唯一确定：

  q(u1)
  q(u2)

  各自是多少。

  为什么不能唯一确定
  例如：

  q(e1)=1, q(e2)=1

  下面是两个球面，总和为 2。

  经过 k=2,l=2 后，上方总和仍是：

  q(u1)+q(u2)=2

  可能是：

  q(u1)=1, q(u2)=1

  两个球面；

  也可能是：

  q(u1)=2, q(u2)=0

  一个是两个球面等价的总量分配，另一个是环面分量等组合。仅凭树结构无法判断哪条出边拿到多
  少。

  实际处理建议

  - 如果只需要总亏格变化：k=2,l=2 的总贡献为 0。
  - 如果要给每条出边标注亏格：必须借助额外信息，例如等值面连通分量追踪、seed set、网格三角
    片归属，或直接在出边中间等值处提取网格算 V-E+F。

  - SoS 中拆分顺序可以是 1-saddle → 2-saddle，也可以因扰动顺序表现为不同虚拟结构；但总贡献
    保持不变。
