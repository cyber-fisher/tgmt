# 两组参数结果简报

## 1. 实验目的

比较不同简化阈值下，ActiveGraph 简化结果的变化情况，验证参数是否已经真实作用于节点筛选过程。

## 2. 参数设置

本次对比使用两组参数：

### `strict`

- `j = 0.05`
- `p = 0.02`
- `b = 0.02`

特点：

- 阈值更严格
- 对差异更敏感
- 更容易保留中间节点

### `loose`

- `j = 0.8`
- `p = 0.5`
- `b = 0.5`

特点：

- 阈值更宽松
- 只有差异较大时才保留节点
- 简化程度更高

## 3. 结果对比

输入树节点总数为 `593`。

两组参数的保留结果如下：

- `strict`：保留 `436` 个节点，约占 `73.5%`
- `loose`：保留 `260` 个节点，约占 `43.8%`

可以看到：

- 严格参数下，保留节点明显更多
- 宽松参数下，删除了更多链上的中间节点
- 说明 `-j / -p / -b` 三个阈值已经真实影响了最终结果

## 4. 统计文件现象

两组结果的统计文件行数相同：

- `strict_stats.csv`：`444` 行
- `loose_stats.csv`：`444` 行

其中：

- 第 1 行是表头
- 后续 `443` 行表示链上候选节点的逐次比较记录

这说明两组实验比较的是同一批候选节点，只是由于阈值不同，`keep` 判定发生了变化。

## 5. 典型现象

从统计文件开头可以看到，同一个候选节点在不同参数下会出现不同结论。

例如：

- 在 `strict` 中，`chain 0` 的节点 `151` 被保留
- 在 `loose` 中，同一个节点 `151` 没有被保留

这说明当前实现已经能够根据差异阈值调整链上节点的保留密度。

## 6. 输出文件

本次结果文件位于 `result/` 目录：

- `strict_simplified_nodes.vtu`
- `strict_simplified_arcs.vtu`
- `strict_stats.csv`
- `loose_simplified_nodes.vtu`
- `loose_simplified_arcs.vtu`
- `loose_stats.csv`

## 7. 结论

本次实验表明：

- 参数调节已经生效
- `strict` 参数会得到更细的简化结果
- `loose` 参数会得到更粗的简化结果

因此，当前程序已经具备通过阈值控制简化强度的能力，后续可以继续围绕 `j / p / b` 做参数调优。
