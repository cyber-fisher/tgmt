# 活性子图向量化方案

## 一、问题背景

原来的做法是直接比较两个活性子图的边集是否相似。也就是说，把两个 merge tree 节点对应的局部活性区域都提取出来，然后看它们包含的激活边是否重合、重合多少。

这种方法直观、容易实现，但有一个明显问题：它对局部位置偏移和轻微结构扰动非常敏感。

例如，两个子图在整体形态上可能非常接近，只是局部位置发生了一点偏移，或者少数边因为数值扰动没有被激活。此时从人的角度看，它们代表的局部结构并没有发生本质变化，但边集相似度可能会明显下降。这样会导致 merge tree 简化过程过于保守，保留很多其实变化不大的节点。

因此，更合适的思路是：

> 不直接比较两个子图的边集，而是先把每个活性子图编码成一个固定长度的向量，再在向量空间中比较它们的相似度。

这样做的核心目标是把“边是否完全重合”变成“局部结构是否相似”。向量表示可以允许轻微扰动，同时保留真正重要的结构差异，从而更稳健地判断两个 merge tree 节点之间是否发生了值得保留的变化。

---

## 二、统一框架

不论后面采用哪一种编码方式，整体流程都保持一致。也就是说，方案 A 和方案 B 的差异只在“如何把活性子图编码成向量”这一步，其他模块应该尽量复用。

首先，将 vti 对应的原始规则网格看成一张全局图：

- 图的顶点对应原始网格点。
- 图的边对应网格邻接关系，第一版建议采用 6 邻接。
- 每个顶点带有标量值、空间坐标和 `SegmentationId`。

然后，对于 merge tree 上的每个节点 `n`，执行以下步骤：

1. 取该节点的标量值作为阈值：`t = Scalar(n)`。
2. 在原始全局图中，提取跨越阈值 `t` 的活性边和活性顶点。
3. 加入局部分割区域限制，只保留与当前 merge tree 节点相关的局部区域。
4. 将得到的活性子图 `G_n` 编码成固定长度向量 `z(n)`。
5. 用向量之间的余弦相似度代替原来的边集相似度。
6. 根据相似度结果决定当前节点是否可以在简化过程中跳过。

整体流程可以概括为：

```text
merge tree 节点 n
    |
    |  取节点标量值 t = Scalar(n)
    v
从原始规则网格中提取活性子图 G_n
    |
    |  方案 A：谱投影 + 统计特征
    |  方案 B：GNN 编码器 + 图级池化
    v
固定长度向量 z(n)
    |
    |  与参考节点向量比较
    v
余弦相似度 sim
    |
    v
merge tree 简化决策
```

这种设计有一个工程上的好处：只要先把子图提取、向量接口、相似度比较和简化决策做成统一管线，后续就可以很方便地替换编码器。先跑方案 A，之后换成方案 B 时，不需要重写整个系统。

---

## 三、方案 A：谱投影 + 统计特征

方案 A 是基础版本，适合作为第一阶段的工程基线。它的目标不是最强表达能力，而是先让整条管线稳定跑通，并得到一个可解释、可对比的结果。

推荐流程是：

> 原始规则网格统一建图 → 局部活性子图提取 → 谱投影嵌入 → 统计特征拼接 → 余弦相似度比较

具体步骤如下。

### 3.1 计算全局谱基底

先在原始全局图上计算一组统一的图谱基底。可以理解为在整个规则网格上建立一套公共坐标系，之后每个局部活性子图都投影到这套坐标系里。

为了控制计算量，实际实现时不需要保留全部谱基底，只需要保留前 `k` 个低频基底。例如可以从 `k = 32`、`64` 或 `128` 开始实验。

### 3.2 构造子图信号

对于每个活性子图 `G_n`，在全局图顶点上构造一个信号。最简单的做法是二值掩码：

- 如果顶点属于当前活性子图，则信号值为 1。
- 如果顶点不属于当前活性子图，则信号值为 0。

也可以使用更细的信号，例如顶点标量值、与阈值的距离，或者活性强度。但第一版建议先用二值掩码，便于验证。

### 3.3 谱投影

将子图信号投影到全局谱基底上，得到一个长度为 `k` 的谱投影向量。这个向量描述了当前子图在全局谱空间中的分布特征。

谱投影的优点是固定长度、实现清晰，并且所有子图都共享同一套基底，方便直接比较。

### 3.4 拼接统计特征

仅靠谱投影可能不足以描述局部结构，因此建议再拼接一组简单统计特征，例如：

- 活性顶点数。
- 活性边数。
- 连通分量数。
- 最大连通分量大小。
- 局部质心坐标。
- 包围盒大小。
- 子图空间尺度。
- 分割标签直方图。

最终向量可以写成：

```text
z(n) = concat(谱投影向量, 统计特征向量)
```

在计算余弦相似度之前，建议对各类特征做归一化，避免某一类数值范围过大的特征主导相似度。

### 3.5 方案 A 的优点和局限

方案 A 的优点是：

- 实现难度低。
- 结果比较可解释。
- 不需要训练数据。
- 适合作为第一版 baseline。

它的局限也比较明显：

- 谱基底是在全局图上计算的，对很细的局部结构区分能力有限。
- 两个局部形态不同的小子图，投影到低维全局谱空间后可能变得很接近。
- 统计特征是粗粒度摘要，可能无法区分细节拓扑差异。
- 如果两个子图的顶点数、边数、包围盒等统计量接近，但内部连接方式不同，方案 A 可能会误判为相似。

因此，方案 A 更适合作为“能跑通、能对比、能解释”的基础版本，而不是最终表达能力最强的方案。

## 四、方案 B：GNN 编码器

方案 B 是升级版本。它的核心思想是：不再把局部子图投影到全局谱基底上，而是直接在局部子图内部做邻域聚合，让模型从子图结构中自动提取表示。

GNN 的优势在于，它天然适合处理图结构。每个顶点先根据自己的特征生成表示，然后通过图边从邻居收集信息。经过若干层聚合后，每个顶点的表示就包含了周围局部拓扑信息。最后再把所有顶点表示池化成一个图级向量，作为整个活性子图的表示。

### 4.1 节点特征设计

活性子图中的每个顶点都需要一个输入特征向量。建议第一版包含以下内容：

- **标量值**：建议归一化到 0 到 1。
- **与阈值的距离**：例如 `scalar - t` 或 `abs(scalar - t)`，用于描述该点离当前等值阈值有多远。
- **中心化空间坐标**：用顶点坐标减去当前子图所有活性顶点的坐标均值。
- **局部度数**：该顶点在活性子图中连接了多少条边。
- **分割标签**：`SegmentationId`，可以使用 one-hot，也可以使用 embedding。

其中，中心化空间坐标非常重要。它让编码器关注子图内部的相对空间关系，而不是绝对位置。

例如，两个子图形状相同，但一个出现在网格左侧，另一个出现在网格右侧。如果直接使用绝对坐标，它们的向量可能差异很大；如果使用中心化坐标，它们会更容易被判断为相似。这正好符合“对局部位置偏移不敏感”的需求。

为了进一步提高稳定性，还可以考虑把中心化后的坐标除以子图的空间尺度，例如包围盒对角线长度。这样可以减少子图整体大小对向量的影响。

### 4.2 编码器结构

第一版建议使用一个简单的 2 层 GCN：

```text
节点特征 X
    |
    v
第 1 层 GCN：邻域聚合 + 归一化 + 激活函数
    |
    v
第 2 层 GCN：邻域聚合 + 归一化 + 激活函数
    |
    v
图级池化：mean pooling 或 mean + max pooling
    |
    v
固定长度向量 z(n)
```

推荐隐藏维度从 64 开始。如果子图复杂度较高，可以尝试 128；如果数据量大、速度压力明显，可以尝试 32。

### 4.3 为什么先用 GCN

当前图来自规则网格，第一版采用 6 邻接，节点度数比较均匀。因此不一定需要复杂的注意力机制来区分不同邻居的重要性。

GCN 的优点是简单、稳定、计算成本低，适合第一版验证。如果后续发现不同邻居的重要性差异很大，再考虑 GAT、GraphSAGE 或其他更复杂的模型。

### 4.4 为什么先用 2 层

2 层 GCN 可以让每个顶点聚合 2 跳邻域的信息。对于 6 邻接规则网格来说，2 跳邻域已经能覆盖比较典型的局部拓扑模式。

如果层数太少，模型只能看到非常局部的信息，表达能力可能不足。如果层数太多，容易出现过平滑问题，也就是不同顶点的表示越来越接近，反而丧失区分能力。

因此，2 层是一个比较稳妥的起点。

### 4.5 图级池化

GNN 输出的是每个顶点的表示，但最终需要的是整个活性子图的向量。因此需要做图级池化。

最简单的方式是均值池化：

```text
z(n) = mean({h_v | v 属于 G_n})
```

均值池化比较稳定，不太受子图节点数量影响。但它可能会抹掉少数显著局部结构。为了提高区分度，可以把均值池化和最大池化拼接：

```text
z(n) = concat(mean_pool(H), max_pool(H))
```

如果再希望保留子图规模信息，可以额外拼接 `log(活性顶点数)`、`log(活性边数)` 等统计量。

### 4.6 是否需要训练

建议分两步。

第一步，先不训练，把随机初始化的 GNN 当作结构随机投影 baseline。这样做可以快速验证 GNN 管线是否可行，也可以与方案 A 做初步对比。

不过需要注意：随机初始化 GNN 的稳定性不一定足够，它的结果会受到随机种子、特征尺度和归一化方式影响。因此它更适合作为实验基线，而不应该直接视为最终可靠方案。

第二步，如果未训练 GNN 的效果不够，再加入自监督或对比学习。

对比学习的基本思路是：

- 对同一个活性子图做轻微扰动，构造正样本对。
- 对不同活性子图构造负样本对。
- 训练编码器，让正样本对的向量更接近，让负样本对的向量更远离。

轻微扰动可以包括：

- 随机删除少量边。
- 随机遮掩部分节点特征。
- 加入轻微标量噪声。
- 对坐标做小幅扰动。

这样训练出来的编码器可以学到：哪些变化只是噪声或轻微扰动，应该忽略；哪些变化代表明显结构差异，应该保留。

## 五、节点相似度比较

不论使用方案 A 还是方案 B，每个 merge tree 节点最终都会得到一个固定长度向量 `z(n)`。

对于两个节点 `n_i` 和 `n_j`，使用余弦相似度比较它们的向量：

```text
sim(n_i, n_j) = <z_i, z_j> / (||z_i|| ||z_j||)
```

其中：

- `sim` 越接近 1，说明两个节点对应的局部活性子图越相似。
- `sim` 越低，说明两个节点对应的局部结构差异越明显。

在 merge tree 简化中，可以这样理解：

- 如果相似度高于阈值，说明当前节点相比参考节点没有明显新变化，可以跳过。
- 如果相似度低于阈值，说明当前节点带来了新的局部结构变化，应该保留。

需要注意的是，方案 A 和方案 B 的向量分布可能不同，因此即使都使用余弦相似度，也不一定能直接共用同一个阈值。`0.95` 可以作为初始值，但最好通过实验校准。

## 六、在 merge tree 简化中的使用方式

对于一条拓扑链上的节点序列，可以采用以下策略。

假设链上的节点为：

```text
n_0, n_1, n_2, ..., n_k
```

其中 `n_0` 是起点临界节点，`n_k` 是终点临界节点，中间节点是普通节点。

简化流程如下：

1. 保留起点临界节点 `n_0`。
2. 将 `n_0` 作为当前参考节点 `ref`。
3. 依次遍历后续普通节点 `n_i`。
4. 计算 `sim(z(n_i), z(ref))`。
5. 如果相似度高于阈值，认为 `n_i` 相比 `ref` 变化不大，可以跳过。
6. 如果相似度低于阈值，保留 `n_i`，并将 `ref` 更新为 `n_i`。
7. 最后保留终点临界节点 `n_k`。

伪代码如下：

```text
kept = [n_0]
ref = n_0

for n_i in chain[1 : k]:
    sim = cosine(z(n_i), z(ref))
    if sim < threshold:
        kept.append(n_i)
        ref = n_i

kept.append(n_k)
```

这个策略的含义是：不是拿每个节点和起点固定比较，而是拿它和“上一个被保留的节点”比较。这样可以捕获沿着拓扑链逐渐累积的结构变化。

## 七、边界情况

GNN 方案尤其需要处理一些特殊子图，否则容易出现不稳定结果。

### 7.1 空子图

如果当前节点没有提取到任何活性边或活性顶点，可以直接返回零向量，或者返回一个专门的空子图向量。

第一版建议返回零向量，并在计算余弦相似度时额外处理零范数情况，避免除零。

### 7.2 极小子图

如果子图只有 1 到 3 个节点，GNN 的邻域聚合基本退化，结构信息很少。此时可以直接使用节点特征的均值作为图表示，或者拼接简单统计量。

这样比强行经过 GNN 更稳定。

### 7.3 极大子图

如果子图有上万甚至更多节点，完整 GCN 可能计算较慢。可以考虑：

- 使用 GraphSAGE 做邻居采样。
- 对活性子图进行空间降采样。
- 限制每个 merge tree 节点的局部区域范围。
- 对超大子图使用方案 A 或统计特征作为降级策略。

### 7.4 标签数量过多

如果 `SegmentationId` 类别很多，直接 one-hot 会导致特征维度过高。此时可以使用 embedding，或者只保留局部分割标签直方图作为图级统计特征。

## 八、如何判断新方案是否更好

判断一个编码方案是否更好，不能只看单个相似度值，而应该从稳定性、区分度和最终简化质量三方面评估。

### 8.1 抗扰动性

对同一个活性子图做轻微扰动，例如随机删除 10% 的边，然后比较扰动前后的向量相似度。

好的编码器应该让轻微扰动前后的向量仍然接近，余弦相似度应尽量接近 1。

### 8.2 区分度

选择一些结构明显不同的子图对，比较它们的向量相似度。

好的编码器应该让明显不同的子图对相似度较低，同时让结构接近的子图对相似度较高。

建议画出两类样本的相似度分布：

- 相似子图对的 similarity histogram。
- 不相似子图对的 similarity histogram。

如果两类分布分离明显，说明编码效果较好，也更容易选择简化阈值。

### 8.3 简化质量

在相同输入数据上，对比旧方案、方案 A 和方案 B 的简化结果：

- 保留的 merge tree 节点数量。
- 被删除节点是否确实变化不大。
- 简化后重建结果的误差。
- 关键拓扑结构是否被保留。
- 结果在轻微噪声下是否稳定。

最终应该以简化质量为准。向量相似度只是中间指标，真正目标是得到更合理、更稳定的 merge tree 简化结果。

## 九、参数推荐

| 参数 | 方案 A：谱投影 | 方案 B：GNN |
|------|----------------|-------------|
| 编码维度 | 谱投影维度 + 统计特征维度 | 隐藏维度 64 起步 |
| 谱投影维度 | 32、64 或 128 | 不需要 |
| 网络结构 | 不需要 | 2 层 GCN |
| 节点特征 | 子图掩码或活性强度 | 标量值、阈值距离、中心化坐标、度数、分割标签 |
| 图级 readout | 不需要 | mean pooling，或 mean + max pooling |
| 坐标处理 | 作为统计特征，例如质心和包围盒 | 子图级中心化，必要时尺度归一化 |
| 相似度 | 余弦相似度 | 余弦相似度 |
| 初始简化阈值 | 0.95，可调 | 0.95 起步，但建议重新校准 |
| 是否需要训练 | 不需要 | 第一版不训练，效果不够再做对比学习 |

## 十、实施建议

建议按阶段推进。

第一阶段，先实现统一管线：

```text
子图提取 → 编码器接口 → 向量归一化 → 余弦相似度 → 简化决策
```

这一阶段先不要急着上复杂模型，重点是确认每一步输入输出正确，尤其是活性子图提取和 merge tree 链上简化逻辑。

第二阶段，实现方案 A。它作为 baseline，帮助确认整条管线能跑通，并提供一个可解释的对照结果。

第三阶段，把第 4 步的编码器替换成方案 B 的 GNN 编码器。其他模块尽量不动，这样可以公平比较两种编码方式对简化结果的影响。

第四阶段，根据实验结果决定是否训练 GNN。如果未训练 GNN 已经明显优于方案 A，可以先停在这里。如果效果不稳定、阈值难选，或者抗扰动性不足，再引入对比学习。

推荐的落地顺序是：

```text
1. 跑通方案 A
2. 固定子图提取和简化逻辑
3. 接入未训练 GNN 作为 baseline
4. 对比 A 和 B 的简化结果
5. 校准相似度阈值
6. 必要时加入对比学习训练
```

这样推进的好处是风险比较低。每一步都能得到可对比的中间结果，也便于定位问题到底来自子图提取、编码器，还是简化策略本身。
