# 拓扑-几何合并树简化项目汇总整理

本文档整理当前目录 `tgmt` 下的主要材料，参考 `思路.md` 的组织方式，将研究目标、数据来源、三类算法路线、工程实现、实验结果、可视化产物和后续工作统一汇总。目录中存在大量 `.vtu`、`.vti`、`.vtp`、`.csv`、`.txt`、`.py` 和 `.md` 文件，其中二进制/可视化结果文件不逐一展开，而按算法与实验阶段归类说明。

## 1. 项目目标

本项目围绕合并树或增广合并树的节点简化展开，核心目标是：

> 在严格保留拓扑临界节点的前提下，对每条拓扑边上的大量普通节点进行几何或结构去冗余，只保留能够代表显著形态变化的关键节点。

合并树上的一条拓扑链可记为：

```text
S_start, S_1, S_2, ..., S_n, S_end
```

其中：

- `S_start` 和 `S_end` 是临界节点或结构节点，必须保留。
- 中间节点 `S_i` 是候选普通节点，需要根据几何或结构变化决定保留/删除。
- 简化策略整体采用顺序贪心：保留链起点，以最近一次保留节点作为参考节点，依次比较后续候选节点；若差异超过阈值则保留，并更新参考节点；链终点强制保留。

项目提出并实现了三条主要路线：

| 路线 | 方法名称 | 核心比较对象 | 主要优点 | 主要局限 |
|---|---|---|---|---|
| 思路一 | ISM 等值面相似性 | 局部等值面距离场、互信息、质心、包围盒 | 几何表达较完整，能捕捉形状变化 | 计算较重，参数多 |
| 思路二 | 水平集体积 IoU | 子水平集体积或局部体积变化 | 快速、稳定、易解释 | 只看体积，可能漏掉形状变而体积不变的情况 |
| 思路三 | 原始网格活性子图 | 活性边集合、质心、包围盒、图向量 | 更贴近原始网格结构，可扩展到图学习 | 实现和调参成本更高 |

### 3.1 TTK/MergeTree 三口输出

根据 `构建过程记录.md`，当前流程使用 TTK `standalone/MergeTree/main.cpp` 或相关程序处理标量场，得到合并树/增广合并树三口输出：

| `tangaroa_port_2.vti` | `vtkImageData` | 标量场 `v/u/w`，`SegmentationId`, `RegionSize`, `RegionSpan`, `RegionType`, `Spacing`, `Origin`, `Extent` | 原始体数据和分割标签；用于计算体积、提取等值面近似、枚举活性边。 |

### 3.2 统一处理流程

三条算法虽然度量不同，但整体管线一致：

```text
输入 VTU/VTI 或 tree_output.txt
        |
        v
读取节点、弧、体数据
        |
        v
构建树邻接关系
        |
        v
抽取拓扑链/branch
        |
        v
对每条链做顺序贪心筛选
        |
        v
输出 KeepNode 掩码、简化弧、统计 CSV/TXT、ParaView 可视化文件
```

### 3.3 拓扑链抽取原则

链的端点通常是结构节点。结构节点可由以下规则识别：

- 根节点或叶节点。
- 入度或出度不等于 1 的节点。
- `CriticalType != 0` 的临界节点。

链内节点按 `Scalar` 从小到大排序。链端点必须保留，中间候选节点由具体差异度量决定。

## 4. 思路一：基于 ISM 的等值面相似性比较

### 4.1 方法核心

ISM 方法的目标是不直接硬比较两个等值面三角网格，而是将等值面转为更稳定的空间描述。当前工程版使用局部边界/距离场近似：

```text
节点标量阈值
    -> 局部分割区域裁剪
    -> 边界掩码
    -> 距离场
    -> 互信息/质心/包围盒比较
```

两个节点的主要差异项包括：

- `distIsm = 1 - NMI`：基于距离场直方图的归一化互信息距离。
- `distPos`：边界质心距离或相对位移。
- `distBbox`：包围盒对角线相对变化。

当前简化判据采用 OR 形式：

```text
distIsm >= epsilonIsm
or distPos >= epsilonPos
or distBbox >= epsilonBbox
```

只要任一项超过阈值，就保留候选节点。

## 5. 思路二：基于水平集体积的快速 IoU 比较

### 5.1 方法核心

IoU 方法基于同一拓扑链上子水平集的嵌套关系。若两个阈值对应体积分别为 `V_A` 和 `V_B`，则体积 IoU 可以近似为：

```text
IoU = V_min / V_max
Dist_vol = 1 - V_min / V_max
```

简化判据：

```text
1 - V_min / V_max >= eps
```

若体积变化超过阈值，则保留当前节点。

为了避免“体积相近但形状变化大”的漏检，可以加入轻量补偿项，例如包围盒对角线、质心位移、轴向尺度等。当前目录里的 IoU 主实验主要体现了体积阈值 `eps` 对保留数量的影响。

## 6. 思路三：原始网格活性子图结构相似性

### 6.1 方法核心

活性子图方法直接在原始网格上描述等值面穿过了哪些网格边。对节点 `n`：

```text
t = Scalar(n)
```

原始规则网格中一条边 `(u, v)` 若满足：

```text
f(u) <= t < f(v)
or
f(v) <= t < f(u)
```

则该边为活性边。活性边及其端点构成节点 `n` 的活性子图 `G_n`。

实际实现中，为避免引入其他分支区域，需使用当前节点相邻弧的 `SegmentationId` 做局部限制。即只保留至少一个端点属于相关分割区域的活性边。

### 6.2 第一版：边集 Jaccard + 位置补偿

第一版主判据是活性边集合的 Jaccard 距离：

```text
J(A, B) = |E_A intersect E_B| / |E_A union E_B|
DistJ = 1 - J
```

同时加入轻量几何项：

```text
DistPos  = ||centroid_A - centroid_B|| / max(diag_A, eps)
DistBbox = |diag_A - diag_B| / max(diag_A, eps)
```

当前 C++/工程文档中推荐并使用 OR 判据：

```text
distJ >= epsilonJ
or distPos >= epsilonPos
or distBbox >= epsilonBbox
```

## 7. 活性子图向量化与 GNN 路线

### 7.1 为什么需要向量化

直接比较活性边集合容易受到局部位置偏移、轻微扰动、少数边缺失的影响。为提高鲁棒性，`alg3_graph/活性子图向量化方案.md` 提出了：

```text
活性子图 -> 固定长度向量 -> 余弦相似度/距离 -> 简化决策
```

向量化目标是把“边是否完全重合”转为“局部结构是否相似”。

### 7.2 方案 A：谱投影 + 统计特征

方案 A 是可解释 baseline：

```text
原始规则网格统一建图
    -> 计算全局谱基底
    -> 将活性子图掩码投影到谱基底
    -> 拼接统计特征
    -> 余弦相似度
```

统计特征可包括：

- 活性顶点数。
- 活性边数。
- 连通分量数。
- 最大连通分量大小。
- 质心坐标。
- 包围盒大小。
- 子图空间尺度。
- 分割标签直方图。

优点是不需要训练，易解释；缺点是低维谱投影可能损失局部细节。

### 7.3 方案 B：GNN 编码器

当前已有实现位于：

```text
alg3_graph/gnn/GNN/
```

主要脚本：

| 文件 | 作用 |
|---|---|
| `vectorize_active_subgraphs.py` | 提取每个 merge tree 节点的局部活性子图，用 2 层 GCN 编码为向量，输出 embedding 和链上距离。 |
| `train_gnn_encoder.py` | 使用自监督对比学习训练 GCN 编码器。 |
| `export_paraview_results.py` | 将 GNN 保留结果导出为 ParaView 可视化 VTI/VTU/CSV。 |
| `export_txt_paraview_no_torch.py` | 面向 TXT 数据集的无 torch 导出辅助脚本。 |

节点特征包括：

- 标量值归一化。
- 与阈值的距离。
- 中心化、尺度化后的空间坐标。
- 活性子图内局部度数。
- `SegmentationId` 的 sin/cos 编码。

编码器结构：

```text
节点特征
  -> GCN layer 1 + ReLU
  -> GCN layer 2 + ReLU
  -> mean pooling + max pooling
  -> 拼接 log(活性点数), log(活性边数)
  -> L2 normalize
```

若 hidden dimension 为 `H`，最终向量维度为：

```text
2 * H + 2
```

例如 `hidden=32` 时输出 66 维向量。

### 7.4 GNN 距离与输出

距离定义：

```text
distGNN = 1 - cosine(z_ref, z_candidate)
```

含义：

- 越接近 0，两个活性子图越相似。
- 越大，局部结构差异越明显。
- `distGNN >= epsilon` 时保留候选节点。

输出文件：

| 文件 | 内容 |
|---|---|
| `*_embeddings.csv` | 每个 merge tree 节点的 embedding。 |
| `*_chain_distances.csv` | 沿拓扑链的 `refNodeId/candidateNodeId/distGNN/keep`。 |
| `*_volume_for_contour.vti` | ParaView Contour 使用的体数据，保留原始标量并添加 GNN 标记数组。 |
| `*_kept_nodes.vtu` | GNN 保留节点点云。 |
| `*_kept_active_edges.vtu` | GNN 判定为变化明显的候选节点对应活性边。 |
| `*_contour_values.csv` | 保留节点对应的 scalar 值，便于 ParaView 手动输入等值面。 |

### 7.6 GCN 训练方案与结果

训练脚本：

```text
GNN/train_gnn_encoder.py
```

训练目标：

```text
同一个活性子图的两次随机增强 -> embedding 更近
不同活性子图 -> embedding 更远
```

正样本增强：

| 增强方式 | 推荐参数 | 说明 |
|---|---:|---|
| edge dropout | 0.10 | 随机删除少量活性边。 |
| feature mask | 0.10 | 随机遮挡节点特征。 |
| scalar noise | 0.01 | 标量相关特征加轻微噪声。 |
| coord noise | 0.01 | 中心化坐标加轻微噪声。 |

损失函数：

- InfoNCE / NT-Xent。
- batch 内其他图作为负样本。
- 推荐 `temperature=0.2`。

当前 `gnn_encoder_dataset_txt_loss.csv` 训练日志：

| epoch | loss | pos_sim | neg_sim |
|---:|---:|---:|---:|
| 1 | 4.0646 | 0.9993 | 0.9819 |
| 200 | 2.2475 | 0.8476 | 0.3131 |

结论：

- loss 明显下降。
- 负样本相似度从 0.9819 降至 0.3131，区分度明显增强。
- 正样本仍保持较高相似度，说明训练基本有效。

TXT case17 的训练后链距离结果：

| 文件 | 候选比较 | `keep=1` | `distGNN` 范围 |
|---|---:|---:|---|
| `case17_gnn_trained_eps015_chain_distances.csv` | 376 | 36 | 0 到 1 |
| `case17_gnn_trained_chain_distances.csv` | 376 | 36 | 0 到 1 |

当前限制：

- GNN 仍处于实验 pipeline 阶段，尚未完全替代 C++ 简化器。
- `epsilon` 不能直接沿用 Jaccard 阈值，需要按 `distGNN` 分布重新选。
- 单数据集训练可能泛化不足，后续应增加多 case 训练。

## 9. 方法选择建议

### 9.1 若目标是先打通系统

优先使用 IoU 方法：

- 实现简单。
- 参数解释清楚。
- 计算速度快。
- 结果随 `eps` 单调变化，便于调参。

推荐从 `eps=0.35 ~ 0.55` 开始做 Tangaroa 对比。

### 9.2 若目标是展示几何保真

优先使用 ISM 方法：

- 能更好描述距离场和形状变化。
- 适合解释“为什么这个节点代表几何关键帧”。
- 适合做论文/报告中的高保真方案。

推荐使用宽阈值扫描结果作为主图：

- `very_strict`
- `strict`
- `loose`
- `very_loose`

### 9.3 若目标是结构创新或后续深挖

优先使用活性子图/GNN 方法：

- 直接联系原始网格结构。
- 可从 Jaccard 扩展到谱特征、图编辑距离、GNN、对比学习。
- 更适合作为“拓扑-几何-图表示”结合的创新方向。

短期建议：

1. 先固定 ActiveGraph 的 Jaccard + 位置补偿结果。
2. 用 GNN 向量化作为分析工具，不急着完全替换 C++ pipeline。
3. 逐步验证 GNN 保留节点是否落在 ParaView 中人眼可解释的结构变化区域。

## 10. 当前主要问题与风险

| 问题 | 影响 | 建议 |
|---|---|---|
| 不同方法结果口径不完全一致 | 例如 GNN 的 `kept nodes` 包括结构节点和候选触发节点，不能直接与 C++ 完整简化结果混比。 | 在后续报告中明确“候选触发保留”和“最终节点总数”的定义。 |
| 参数尺度不统一 | ISM、IoU、Jaccard、GNN 距离的数值分布不同。 | 每种方法单独做距离分布统计，再选 p90/p95 或经验阈值。 |
| ISM 中存在极大 `distBbox` 值 | 一些空/极小局部块可能导致除以很小的 bbox 对角线。 | 加强空子图、小 bbox、零范数保护，并在统计中标记异常原因。 |
| IoU 只看体积 | 对形状变化但体积近似不变的情况不敏感。 | 加入质心、bbox、表面积近似或与 ActiveGraph 混合。 |
| ActiveGraph Jaccard 对局部扰动敏感 | 轻微位置偏移可能导致边集重合下降。 | 使用谱投影/GNN embedding 作为更鲁棒的相似度。 |
| GNN 训练数据有限 | 单 case 或少量 case 训练可能泛化不足。 | 扩展 dataset_txt，多 case 训练，并做跨 case 验证。 |

## 11. 推荐后续工作

### 11.1 统一实验输出格式

建议每种方法都统一输出：

```text
method
dataset
params
original_nodes
kept_nodes
original_arcs
simplified_arcs
branches
candidate_comparisons
candidate_kept
runtime
```

这样便于自动汇总成总表。

### 11.2 增加距离分布统计

对每种方法输出距离分布：

- min
- p25
- p50
- p75
- p90
- p95
- max

特别是 GNN，训练后必须重新根据 `distGNN` 分布选阈值。

### 11.3 做统一可视化对比

建议每种方法至少固定 2 到 4 组代表参数，并输出：

- `KeepNode` 着色结果。
- 简化树骨架。
- 典型 branch 的保留节点序列。
- 对应 scalar 值下的等值面。
- 若是 GNN/ActiveGraph，叠加活性边或活性点区域。

### 11.4 建立混合判据

后续可以考虑：

```text
distFinal =
  alpha * distVol
+ beta  * distISM
+ gamma * distGraph
+ delta * distBBox
```

或者采用两阶段策略：

1. IoU 快速初筛，删除明显冗余节点。
2. ISM 或 ActiveGraph/GNN 对剩余候选做精筛。

### 11.5 报告写作建议

可以按以下逻辑组织正式报告：

1. 问题背景：合并树增广节点太多，普通节点需要几何/结构去冗余。
2. 统一框架：按拓扑链保留端点，中间节点顺序贪心筛选。
3. 方法一：ISM 距离场互信息，强调高保真。
4. 方法二：IoU 体积法，强调快速、可解释、baseline。
5. 方法三：活性子图与 GNN，强调原始网格结构和可扩展性。
6. 实验：Tangaroa 593 节点，case17/case20 TXT 数据。
7. 对比：压缩率、参数敏感性、可视化质量、局限。
8. 后续：混合判据、自监督 GNN、多数据集验证。
