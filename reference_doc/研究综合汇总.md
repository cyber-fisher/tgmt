# 基于拓扑-几何合并树的节点简化研究

核心目标是：

> **在保留合并树关键拓扑结构的前提下，从增广合并树的大量普通节点中筛选出真正代表几何或结构变化的节点，从而减少冗余，并保留主要等值面演化过程。**

## 1. 问题背景

合并树用于描述标量场中连通分量随标量值变化而产生、合并和消亡的过程。临界节点对应重要拓扑事件，树上的弧或拓扑边表示两个拓扑事件之间的连续演化。

实际数据中，为了更细致地记录等值面变化，合并树常被增广，在临界节点之间插入大量普通节点。这些普通节点可以记录等值面沿标量方向的变化，但也带来明显冗余：

- 树结构过密，主要拓扑分支不够突出。
- 相邻普通节点对应的等值面形态可能非常相似。
- 可视化时会出现大量重复或近似的等值面。
- 后续分析、存储和交互成本增加。

因此，本文关注的不是删除临界节点，而是：

> 在保留临界节点和主要拓扑结构的前提下，对每条拓扑边内部的普通节点进行几何或结构去冗余。

## 2.研究动机

### 几何角度

从几何角度看，增广合并树中的普通节点可以理解为等值面演化过程中的采样点。沿着一条拓扑边，标量阈值逐渐变化，对应的等值面或等值区域也连续变化。

**但并不是每个普通节点都对应明显不同的几何状态。很多相邻节点只表现为等值面的轻微移动、平滑膨胀或局部小扰动。这些节点在几何上高度相似，保留下来会造成冗余。**

另一方面，只保留临界节点又过于粗糙。因为两个拓扑事件之间虽然没有新的连通性变化，但等值面形状可能发生明显变化。例如：

- 等值面体积明显变化。
- 形状从紧凑变成长条。
- 局部出现细颈、延展或空间偏移。
- 等值面穿过了新的原始网格区域。

这些变化未必产生新的拓扑事件，但对理解标量场结构仍然重要。

因此，从几何角度出发，我们希望在每条拓扑边内部选出少量“几何代表节点”，用较少节点表达主要形态变化。

### 拓扑角度

从拓扑角度看，合并树的核心语义来自临界节点和分支结构。极小值、极大值、鞍点、根节点、叶节点和分支节点不能被随意删除，否则会破坏合并树对标量场连通性演化的表达。

因此，简化不能只根据几何相似性在全局范围内做。不同拓扑分支上可能存在形状相似的等值面，但它们在合并树中的语义不同，不能互相替代。

拓扑结构在这里提供三个约束：

1. 临界节点和结构节点必须保留。
2. 普通节点的筛选应限制在同一条拓扑链内部。
3. 删除普通节点后，需要重建简化树，使结果仍是一棵合法的合并树。

可以概括为：

```text
拓扑结构决定哪些节点不能删
拓扑链决定哪些节点可以比较
几何度量决定链内普通节点是否保留
```

## 3.比较对象：不是全局等值面

这里需要明确一个关键点：

> 比较两个节点时，不应该提取同一标量值下的全局所有等值面，也不是把所有等值面取平均后比较。

原因是，同一个标量值 `t` 下，标量场中可能存在多个不连通等值面，其中一些属于其他拓扑分支或其他链。如果全部拿来比较，就会把无关结构混入当前节点。

更合理的比较对象是：

> 当前节点在所属拓扑链或局部分割区域内诱导出的局部等值结构。

可以记为：

```text
S(v) = {x in Omega(v) | f(x) = f(v)}
```

其中：

- `f(v)` 是节点的标量值。
- `Omega(v)` 是节点对应的局部区域，例如相邻弧的 `SegmentationId`、当前拓扑链区域或局部分支区域。
- `S(v)` 是局部等值结构，而不是全局等值面。

如果使用活性子图，则比较对象是局部区域内被阈值截穿的原始网格边：

```text
E_active(v) = {(u, w) |
  f(u) <= f(v) < f(w) or f(w) <= f(v) < f(u),
  and (u, w) belongs to Omega(v)}
```

这样可以避免其他不连通链上的等值面干扰当前比较。

从当前代码看，需要更谨慎地表述为：

- ISM 和 GNN/ActiveGraph 的“节点筛选过程”确实是按拓扑链进行的。
- ISM 和 GNN/ActiveGraph 的“节点描述符构造”不是直接使用整条链区域。VTU/Segmentation 版本主要使用当前节点相邻 arc 的 `SegmentationId` 或类似局部区域；TXT 版本则更多使用 subtree/local region。
- `idea_ism/support_txt/TreeTxtIsmSimplifier.cpp` 里的 TXT ISM 版本，源码上可以确认是按 `branch` 做贪心筛选，但每个节点的 ISM 描述符来自该节点的 `subtreeVertices`。它先用子树点集的包围盒生成二值 mask，再从 mask 边界生成距离场。因此它不是全局等值面比较，也不是只比较当前 branch 上的一小段；如果当前节点的子树包含其他旁支，这些旁支会进入 ISM 描述符。
- `TreeTxtIoUSimplifier.cpp` 里现有的 IoU 代码已经改成按 branch 的局部体积 IoU 原型：它先在 branch 的局部范围内按节点标量构造离散子水平集，再用这些局部体积做 IoU。
  也就是说，当前代码不再把 `subtreeVertices` 当成最终比较对象；这里的最终方法目标仍然是“局部体积 IoU”。

## 4.总体简化框架

三种策略都可以放入同一个链上贪心框架。

对每条拓扑链：

```text
v_0, v_1, ..., v_k
```

其中 `v_0` 和 `v_k` 是结构节点，必须保留。中间节点是候选普通节点。

简化流程：

```text
keep = [v_0]
ref = v_0

for v_i in v_1 ... v_{k-1}:
    计算 ref 和 v_i 的局部等值结构差异 Dist(ref, v_i)
    如果 Dist 超过阈值:
        保留 v_i
        ref = v_i

keep.append(v_k)
```

因此，关键问题变成：

> 如何定义两个链内节点之间的局部几何或结构差异？

当前主要考虑三种策略。

## 策略一：基于 ISM 的局部形状相似性

ISM 方法关注局部等值结构的形状差异。

基本思路是：不直接比较等值面三角网格，而是把局部等值结构转换成距离场或类似的空间描述，再通过互信息衡量两个节点的相似性。

对节点 `v`：

```text
局部区域 Omega(v)
    ↓
阈值 f(v)
    ↓
局部等值边界或等值面
    ↓
距离场描述符
```

两个节点之间可定义：

```text
Dist_ISM = 1 - NMI(D_ref, D_candidate)
```

其中 `NMI` 是距离场之间的归一化互信息。

同时可以加入：

- 质心位移。
- 包围盒尺度变化。

保留判据可以写成：

```text
Dist_ISM >= epsilon_ism
or DistPos >= epsilon_pos
or DistBbox >= epsilon_bbox
```

优点：

- 能捕捉体积相近但形状不同的情况。
- 几何表达较完整。
- 适合强调几何保真。

缺点：

- 计算较重。
- 参数较多。
- 如果局部区域内有多个不连通分量，整体距离场会把它们混在一起，需要后续分量级处理。
- 当前实现需要分版本理解：VTU/Segmentation 版构造单个节点描述符时可能使用该节点所有 incident arc 的 `SegmentationId`，不显式传入当前 chain id；TXT 版 `support_txt` 则使用节点的 `subtreeVertices` 构造局部二值 mask 和距离场。两者的共同点是筛选沿 branch 进行，但描述符本身都不等价于“只取当前 branch 的一小段区域”。

## 策略二：基于局部体积 IoU 的快速比较

IoU 方法关注局部子水平集体积变化。

当前代码实现的是一个 branch-local 的离散近似：它在同一条 branch 上，按当前节点标量从局部节点集合中构造子水平集，再用这个子水平集的规模近似体积进行比较。

对节点 `v`，在局部区域 `Omega(v)` 内计算对应的体积：

```text
V(v) = volume({x in Omega(v) | f(x) <= f(v)})
```

两个节点之间的体积相似性可以写成：

```text
IoU = min(V_ref, V_candidate) / max(V_ref, V_candidate)
```

差异为：

```text
Dist_vol = 1 - IoU
```

当：

```text
Dist_vol >= epsilon_vol
```

则保留候选节点。

这里要特别注意两点：

- 这里的 IoU 目标是几何体积 IoU，所以文中应把它写成“局部子水平集体积变化”。
- 局部区域 `Omega(v)` 需要限制在当前 branch 对应的局部范围内，避免把其他拓扑分支的体积混进来。

优点：

- 实现简单。
- 参数直观。
- 适合作为 branch 内的快速初筛。

缺点：

- 只看体积，不直接看形状。
- 对“体积相近但形态变化明显”的情况不敏感。
- 多分支情况下，如果 `Omega(v)` 选得不够局部，其他旁支的体积会干扰比较。
- 当前代码是局部体积 IoU 的离散原型，若后续要接真实体数据，还需要把 `Omega(v)` 和 `volume` 明确落到体素/分割区域上。

## 策略三：基于原始网格活性子图的结构比较与 GNN 向量化

活性子图方法关注等值面在原始网格中“截穿了哪些边”。

对于原始网格边 `(u, w)`，如果：

```text
f(u) <= t < f(w)
or
f(w) <= t < f(u)
```

则这条边是阈值 `t` 下的活性边。

对节点 `v`：

```text
t = f(v)
```

只在局部区域 `Omega(v)` 中提取活性边，得到活性子图：

```text
G(v) = (V_active(v), E_active(v))
```

### 3.1 活性边集合方法

两个节点可以用活性边集合的 Jaccard 距离比较：

```text
Dist_J = 1 - |E_ref intersect E_candidate| / |E_ref union E_candidate|
```

同时加入：

- 活性子图质心位移。
- 包围盒尺度变化。

保留判据：

```text
Dist_J >= epsilon_j
or DistPos >= epsilon_pos
or DistBbox >= epsilon_bbox
```

这个版本对应 `idea_graph/active_graph_simplifier` 和 `idea_graph/support_txt` 中的 ActiveGraph 实现。它的描述符是显式的活性边集合，优点是直观、可解释，缺点是 Jaccard 对轻微位置偏移较敏感。

### 3.2 GNN 活性子图向量化方法

在 `idea_graph/GNN` 里，这一策略进一步扩展成了 GNN 向量化。核心不是换掉活性子图，而是把同一个局部活性子图编码成固定长度向量，再用向量距离做比较：

```text
局部活性子图
    ↓
节点特征
    ↓
GCN / GNN 编码
    ↓
图级 embedding
    ↓
cosine distance
```

GNN 版和 ActiveGraph 版共享同一个局部区域定义，只是前者把活性子图编码成向量。实践上它仍然是沿 branch 做贪心参考节点更新，只不过距离从 Jaccard 换成了 embedding 的 cosine distance。

当前实现中，GNN 的节点特征包含标量归一化、相对阈值、局部坐标、节点度和 `SegmentationId` 的周期编码；图级向量由 GCN 隐层的 mean pooling、max pooling 和图规模统计拼接得到。训练权重可以通过 `train_gnn_encoder.py` 学习；如果不加载训练权重，则使用确定性随机权重作为 baseline。

优点：

- 更贴近原始离散网格结构。
- 不依赖完整等值面重建。
- 活性边集合版本可解释性强。
- GNN 版本能把局部结构、空间位置、标量和分割信息编码进统一向量。

缺点：

- Jaccard 对轻微位置偏移较敏感。
- GNN embedding 的解释性弱于显式边集合。
- 如果使用未训练的随机权重，GNN 结果更适合作为结构化 baseline，而不是最终学习模型。
- 如果局部活性子图包含多个不连通分量，直接合成一个边集合会掩盖分量之间的差别。
- 当前 GNN/ActiveGraph Python 代码中，链上比较是明确的；VTU 模式下局部区域来自节点 incident `SegmentationId` 的合并 bbox，并进一步用 `SegmentationId` 过滤活性边；TXT 模式下使用 subtree/local region。GNN 版和 ActiveGraph 版共享这个局部活性子图定义，只是前者把子图编码成向量再做 cosine distance。它不是全局等值面，但也不是严格“只取当前 chain 的 arc 区域”，尤其在多 incident arc 节点处需要注意。

## 三种策略之后的统一处理：多分支等值结构

可以把单分支、双分支、三分支、多分支的处理放在三种策略之后，作为一个统一模块。这样逻辑更清楚：

```text
先用三种策略定义“单个局部结构怎么描述和比较”
        ↓
再处理局部区域中存在多个不连通分量时怎么匹配和加权
```

即使已经限制在当前拓扑链或局部分割区域内，某个节点对应的局部等值结构仍然可能包含多个不连通分量。

如果直接把这些分量合成一个整体描述符，会有两个问题：

1. 不同分量之间的对应关系不清楚。
2. 大分量可能主导整体距离，小分量的重要变化可能被淹没。

因此，可以先把局部等值结构分解为多个连通分量：

```text
C(v) = {c_1(v), c_2(v), ..., c_k(v)}
```

每个分量都可以根据前面三种策略构造自己的描述符：

| 策略 | 分量描述符 |
|---|---|
| ISM | 分量局部距离场 |
| IoU | 分量局部体积 |
| ActiveGraph | 分量活性边集合 |
| GNN | 分量活性子图 embedding |

## 5.多分支权重怎么考虑

多分支比较的关键是权重。不能简单平均，因为小噪声分量不应和主分量同等重要；但也不能只按大小加权，否则小而重要的拓扑分支会被忽略。

可以考虑以下几类权重。

### 几何规模权重

对于等值面分量，可以用面积：

```text
w_i = area_i / sum_j area_j
```

对于活性子图，可以用活性边数：

```text
w_i = |E_i| / sum_j |E_j|
```

优点是直观，大结构权重大。缺点是小分支容易被压制。

### 分量体积权重

如果分量对应局部区域，可以使用局部体积：

```text
w_i = volume_i / sum_j volume_j
```

这适合与 IoU 方法和 `RegionSize` 结合。

### 拓扑重要性权重

如果分量对应不同拓扑分支，可以使用拓扑重要性：

```text
w_i = persistence_i / sum_j persistence_j
```

或：

```text
w_i = RegionSpan_i / sum_j RegionSpan_j
```

这样可以避免面积不大但拓扑上重要的分支被忽略。

### 混合权重

更推荐使用混合权重：

```text
raw_w_i = (size_i)^alpha * (topo_i)^beta * (span_i)^gamma
w_i = raw_w_i / sum_j raw_w_j
```

其中：

- `size_i` 可以是面积、活性边数或局部体积。
- `topo_i` 可以是 persistence、RegionSpan 或分支重要性。
- `span_i` 可以是包围盒对角线或空间跨度。

为了避免大分量过度主导，可以取：

```text
alpha = 0.5
```

例如：

```text
raw_w_i = sqrt(|E_i|) * RegionSpan_i
```

这样既考虑几何规模，也考虑拓扑重要性。

## 6.最终方法架构

可以将最终方法理解为两层：

第一层是基础节点差异策略：

```text
ISM：比较局部形状
IoU：比较局部体积
ActiveGraph/GNN：比较原始网格活性子图结构
```

第二层是多分支统一处理：

```text
局部等值结构分解为若干连通分量
        ↓
单分支直接比较
双分支/三分支先匹配再加权
多分支使用最优匹配
分量数不一致时加入未匹配惩罚
```

## 8.总结

本研究的核心是：在拓扑约束下做几何感知的合并树普通节点简化。

拓扑层面，临界节点和分支结构必须保留，普通节点筛选应限制在同一拓扑链内部。几何层面，链内普通节点只有在局部等值结构发生明显规模、形状、位置或原始网格截交结构变化时才需要保留。

三种基础策略分别从不同角度定义节点差异：

- ISM：局部形状相似性。
- IoU：局部体积变化。
- ActiveGraph/GNN：原始网格活性子图结构变化。

在三种策略之后，可以统一处理单分支、双分支、三分支和多分支情况。多分支时先做分类，再根据几何规模、局部体积、拓扑重要性或混合权重进行加权比较。这样既避免全局等值面混入，也避免多个局部分量被简单平均后丢失重要变化。
