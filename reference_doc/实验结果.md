# 实验结果

## 思路一：基于 ISM 的等值面相似性比较

### 1. 实验数据

- 节点文件：`tangaroa_port_nodes.vtu`
- 弧文件：`tangaroa_port_arcs.vtu`
- 分割体数据：`tangaroa_port_2.vti`
- 原始节点总数：`593`

### 2. 实验设置

本次采用四组阈值：

| 组别        | epsilonIsm | epsilonPos | epsilonBbox | 说明   |
| ----------- | ---------- | ---------- | ----------- | ------ |
| very_strict | 0.02       | 0.05       | 0.02        | 很保守 |
| strict      | 0.25       | 1.8        | 0.18        | 保守   |
| loose       | 0.35       | 2.5        | 0.25        | 激进   |
| very_loose  | 0.60       | 5.0        | 0.50        | 很激进 |

### 3. 实验结果

| 组别        | 保留节点数 | 简化强度 |
| ----------- | ---------- | -------- |
| very_strict | 517 / 593  | 最弱     |
| strict      | 465 / 593  | 较弱     |
| loose       | 421 / 593  | 较强     |
| very_loose  | 356 / 593  | 最强     |

### 4. 结果分析

新的阈值配置明显拉大了不同实验组之间的差异。随着阈值逐步增大，节点保留数量持续下降，说明阈值参数能够有效控制简化强度。相比之前仅相差几个节点的情况，这次四组结果已经形成了清晰的梯度，便于后续做可视化对比和参数选择。

## 思路二：基于水平集体积的快速 IoU 比较

### 1. 实验数据与输出

实验目录包含 7 组结果，分别对应：

- `eps = 0.15`
- `eps = 0.25`
- `eps = 0.35`
- `eps = 0.45`
- `eps = 0.55`
- `eps = 0.65`
- `eps = 0.75`

每组结果均包含：

- `output_nodes.vtu`：简化后节点结果
- `output_arcs.vtu`：简化后边结果
- `output_node_mask.vtu`：节点保留掩码
- `output_stats.txt`：统计信息

原始数据在各组实验中保持一致：

- 原始节点数：593
- 原始边数：592

### 2.. 结果汇总

| eps | 保留节点数 | 节点保留率 | 节点压缩率 | 简化后边数 |
| --- | ---: | ---: | ---: | ---: |
| 0.15 | 394 | 66.44% | 33.56% | 393 |
| 0.25 | 367 | 61.89% | 38.11% | 366 |
| 0.35 | 348 | 58.68% | 41.32% | 347 |
| 0.45 | 332 | 55.99% | 44.01% | 331 |
| 0.55 | 316 | 53.29% | 46.71% | 315 |
| 0.65 | 298 | 50.25% | 49.75% | 297 |
| 0.75 | 287 | 48.40% | 51.60% | 286 |

### 3. 结果分析

从整体趋势看，随着 `eps` 增大，保留节点数和简化后边数持续下降，说明更大的阈值会带来更强的结构压缩效果。`eps` 从 `0.15` 提升到 `0.75` 后，保留节点数由 394 降至 287，节点保留率由 66.44% 下降到 48.40%。

从压缩强度看，低阈值下更偏向保留局部细节，高阈值下则更强调主干结构。尤其在 `eps >= 0.55` 后，节点保留率已经接近或低于 53%，说明简化开始进入较强压缩区间。

### 4. 结论

该组实验表明，参数 `eps` 对树结构简化结果具有稳定且单调的控制作用：

- 较小 `eps` 适合保留更多细节信息；
- 较大 `eps` 适合获得更紧凑的骨架表达；
- 若希望在结构保持与压缩率之间取得平衡，`eps = 0.35 ~ 0.55` 是较合适的候选区间。

## 思路三：提取原始网格活性子图并比较结构相似性

### 1. 参数设置

本次对比使用两组参数：

### `strict`

- `j = 0.05`
- `p = 0.02`
- `b = 0.02`

特点：

- 阈值更严格
- 对差异更敏感
- 更容易保留中间节点

### `loose`

- `j = 0.8`
- `p = 0.5`
- `b = 0.5`

特点：

- 阈值更宽松
- 只有差异较大时才保留节点
- 简化程度更高

### 2. 结果对比

输入树节点总数为 `593`。

两组参数的保留结果如下：

- `strict`：保留 `436` 个节点，约占 `73.5%`
- `loose`：保留 `260` 个节点，约占 `43.8%`

可以看到：

- 严格参数下，保留节点明显更多
- 宽松参数下，删除了更多链上的中间节点
- 说明 `-j / -p / -b` 三个阈值已经真实影响了最终结果

### 3. 典型现象

从统计文件开头可以看到，同一个候选节点在不同参数下会出现不同结论。

例如：

- 在 `strict` 中，`chain 0` 的节点 `151` 被保留
- 在 `loose` 中，同一个节点 `151` 没有被保留

这说明当前实现已经能够根据差异阈值调整链上节点的保留密度。
